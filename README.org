#+TITLE: literate-latex
#+OPTIONS: toc:nil

Write code inside a LaTeX document and *load it directly* --- no tangle
step. One =.pamphlet= file is both the source your runtime loads and the
document =pdflatex= typesets. In the spirit of [[https://github.com/jingtaozf/literate-elisp][literate-elisp]] /
[[https://github.com/jingtaozf/literate-lisp][literate-lisp]], with [[https://github.com/daly/axiom][Axiom]]-style ~\begin{chunk}~ syntax.

Direct load works for *Common Lisp* and *Emacs Lisp* (their readers are
programmable). Any language --- or out-of-order assembly --- can use the
optional [[https://www.cs.tufts.edu/~nr/noweb/][noweb]] ~\getchunk~ tangle path instead.

* Quick start

Write =hello.pamphlet=:

#+BEGIN_SRC latex
% -*- mode: literate-latex; literate-lang: lisp -*-
\documentclass{article}
\usepackage{literate-latex}
\begin{document}
\section{Hello}
A greeting.
\begin{chunk}{hello}
(defun hello (name) (format nil "Hello, ~a!" name))
\end{chunk}
\end{document}
#+END_SRC

** Load it in Common Lisp (no tangle)

#+BEGIN_SRC lisp
(load "/path/to/literate-latex/literate-latex.lisp")
(literate-latex:load-pamphlet "hello.pamphlet")
(hello "world")            ; => "Hello, world!"
#+END_SRC

** ...or in Emacs Lisp

Use Emacs Lisp in the chunk (=(format "Hello, %s!" name)=) and load the
package itself with [[https://github.com/jingtaozf/literate-elisp][literate-elisp]] (it is an =.org=, there is no =.el=):

#+BEGIN_SRC elisp
(require 'literate-elisp)
(literate-elisp-load "/path/to/literate-latex/literate-latex.org")
(literate-latex-load "hello.pamphlet")
(hello "world")
#+END_SRC

** Typeset it to PDF

#+BEGIN_SRC sh
TEXINPUTS="/path/to/literate-latex/tex:" pdflatex hello.pamphlet
#+END_SRC

(Or put =literate-latex.sty= where TeX can find it and just run
=pdflatex hello.pamphlet=.)

* Chunk syntax

#+BEGIN_SRC text
\begin{chunk}[load=no]{name}     % optional [opts], required {name}
... code ...                     % loaded verbatim, in document order
\end{chunk}
#+END_SRC

Options (all optional, comma-separated): ~load=no~ skips the chunk,
~lang=elisp~ restricts it to one loader. Same-named chunks concatenate.

* noweb references (optional reorder)

Reference one chunk from another with ~\getchunk{name}~, then assemble
with ~tangle~ --- this is also how you use a pamphlet for a language that
can't load it directly.

#+BEGIN_SRC text
\begin{chunk}{*}
(list
  \getchunk{b}
  \getchunk{a}
  )
\end{chunk}

\begin{chunk}{a}
'a
\end{chunk}

\begin{chunk}{b}
'b
\end{chunk}
#+END_SRC

Tangling the ~*~ root yields ~(list~ / ~  'b~ / ~  'a~ / ~  )~ --- the
helpers are spliced in reference order, indentation kept.

#+BEGIN_SRC lisp
(literate-latex:tangle "demo.pamphlet" "demo.lisp")   ; root defaults to "*"
;; or in Emacs:  (literate-latex-tangle "demo.pamphlet" "demo.lisp")
#+END_SRC

References stand on their own line; indentation is preserved and
compounds through nesting. Reference cycles and undefined names error.

* Editing in Emacs

=.pamphlet= files open in ~literate-latex-mode~ (chunk font-lock, imenu,
folding) on top of ~latex-mode~:

| Key       | Action                            |
|-----------+-----------------------------------|
| ~C-c C-j~ | jump to the chunk under ~\getchunk~ |
| ~C-c C-c~ | typeset to PDF (latexmk)          |
| ~C-c C-p~ | inline preview (xenops/AUCTeX if installed) |
| ~C-c C-l~ | load this pamphlet                |

Inline math/image preview reuses [[https://github.com/dandavison/xenops][xenops]] or [[https://www.gnu.org/software/auctex/][AUCTeX]]'s =preview-latex=
when installed; the PDF + SyncTeX path needs only =pdflatex= / [[https://ctan.org/pkg/latexmk][latexmk]].

* Notes

- Direct load is Common Lisp and Emacs Lisp only; other languages tangle.
- For Emacs Lisp put =lexical-binding: t= in the first-line modeline.
- In the CL reader ~\~ is a macro char, so single-escape symbols (=a\ b=)
  aren't supported --- use =|a b|=. Strings and =#\\= are unaffected.
